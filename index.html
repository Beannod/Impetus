📖 How to use the Report Wizard

        
    

    
Builds SQL for dbo.ReportDefinition. Every script runs as a
       preview first (@commit = 0 inserts inside a transaction,
       shows the stored rows, then ROLLBACKs). Set @commit = 1 to actually persist.


    
Step 1 · Category

    
    
        
  • Paycheck (or any non-Invoice) → report with subreports.
    • 
        
  • Invoice → parent report only; the subreport section is hidden
            and no ReportDefinitionSubReportDefinition link is inserted.
    • 
    


    
Step 2 · Choose mode

    
    
        
  1. Copy existing — clone a report by its source title + category into a
            new title. Subreports are discovered and cloned automatically; names are rewritten
            Avionte → CUO, SPs normalized to CUO_…, and parent↔sub links recreated.
    2. 
        
  2. Create report — supply the parent title, stored procedure,
            SP SQL and ReportDefinitionXML. Paste by hand, or
            📁 Browse folder to auto-fill everything. Add subreports as needed
            (hidden when category = Invoice).
    3. 
    


    
Browse-folder layout

    
Browsing auto-fills the same fields you'd paste into:

    
<ReportFolder>/
  Parent.xml / .rdl          ← parent report (root-level)
  subreports/                ← each child report (any subfolder)
     ChildA.xml
  sp/                        ← stored procedures (filename = SP name)
     CUO_exp_Foo.sql

    
The parent is the file named/pathed parent, else the shallowest XML.
       SP for each report is guessed by best name match (else the SP named in the XML); always editable.
       A Files detected table shows how each file was classified.


    
Buttons

    
    
        
  • Generate Report SQL — the ReportDefinition INSERT (preview/commit).
    • 
        
  • Generate SP Script — a standalone deployment
            script of just the stored procedures (parent + subs).
    • 
        
  • Copy Output — copies the Output SQL box to the clipboard.
    • 
    


    
Workflow

    
    
        
  1. Pick category → mode → fill the fields (or browse).
    2. 
        
  2. Generate Report SQL, copy it, run in SSMS with @commit = 0 to preview.
    3. 
        
  3. Review the returned rows/links, then set @commit = 1 and re-run to persist.
