How to use Report Migration Builder
Quick reference for building a migration script end-to-end

1 Fill in the Source Report

Enter Report Name, Stored Procedure Name, and Build ID. These cascade everywhere — into the read-only Report Details mirror, the Live Preview, and the generated SQL.

Tip: The fields with the Auto badge are filled automatically from the Source Report — don't type into them.

2 Pick a Migration Type

  • Copy from Standard Report — clone an existing report. You'll be asked for an Original Report Name.
  • Create New Custom Report — build from scratch with custom flags, data sources, columns.

3 Configure Parameters

Each row needs both a Param Name and a DB Parameter Name. Description is optional.

  • Add Parameter — append an empty row.
  • Paste from Spreadsheet — paste tab-separated rows from Excel / Google Sheets. Duplicates are skipped automatically.
  • Clear All — when duplicates exist, only the duplicate rows are removed. Otherwise it clears every non-protected row. BuildId and UserId are always kept.
  • Red borders mean the cell is empty or duplicates another row. Hover to see why.

4 Optional: Report Columns & Advanced

Toggle the Report Columns switch to add a column table — each row needs Display Name and Field. Toggle Custom Data Sources & Parameters for additional rptDataSources hookups.

5 Watch the Live Preview

  • Metadata — the computed URL slug, UserClaim, ReportAccess name, etc.
  • Parameters — the current parameter rows as they'll appear in SQL.
  • Access — values used by the optional Report Access script.
  • SQL Preview — click the Preview button in the footer to render the full SQL.
  • The Copy button copies the active tab to your clipboard.

6 Validate & Generate

The footer dot is green when everything's ready and red with a reason when it isn't. The Issues pill in the top-bar shows the total number of issues across the entire form.

  • Validate — preflight check, fires a toast.
  • Preview — renders the SQL into the SQL Preview tab.
  • Generate Migration Script — downloads the SQL file. Blocked while there are unresolved issues.

7 Navigation

The stepper at the top jumps between sections. The active step highlights as you scroll. Press Esc to close this dialog.
